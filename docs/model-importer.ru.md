# Импорт моделей

Импортер моделей **LancerEdit** поддерживает создание файлов **.cmp** и **.3db** (соответственно многосоставные и односоставные модели) из распространённых форматов обмена моделями. Также при необходимости создаются файлы коллизий **.sur**.

Импортер доступен через:
**Tools → Import Model**

---

## Поддерживаемые форматы

### glTF 2.0 (.glb / .gltf)

Формат **glTF 2.0** полностью поддерживается импортером LancerEdit.
Рекомендуется использовать **.glb** или встроенный **.gltf**, чтобы все ресурсы были упакованы в один файл.

Включает:

- текстуры
- пользовательские свойства

---

### Wavefront .obj (.obj)

**Не рекомендуется.**

Ограничения:

- только односоставные модели (.3db)
- нет поддержки пользовательских свойств
- нет поддержки многосоставных моделей

---

### Collada (.dae)

Резервный формат, если glTF 2.0 не поддерживается вашим инструментом.

Ограничения:

- нет пользовательских свойств
- hardpoints и составные соединения не импортируются корректно

---

## Интеграция с Blender

LancerEdit умеет открывать файлы напрямую из **Blender**, автоматически экспортируя их в glTF 2.0.

Если Blender не найден автоматически:

- Откройте **Tools → Options**
- Укажите путь к Blender во вкладке **Import/Export**

---

## Иерархия модели

Требование:

- корневая меш-сетка должна находиться **в корне файла**
- она становится:
  - root частью в .cmp
  - или моделью в .3db

❗ Нельзя помещать её внутрь пустого helper-объекта

Дочерние объекты:

- сохраняются как непосредственные потомки

Проверить структуру можно во вкладке:
**Output Nodes**

---

## Hardpoints (точки крепления)

Hardpoints хранятся как дочерние объекты мешей с пользовательскими свойствами.

Обязательное свойство:

- `hardpoint` — должно быть задано и не равно 0

### Типы

#### Фиксированные (Fixed)

```
hptype = "fix"
```

#### Вращающиеся (Revolute)

```
hptype = "rev"
```

Дополнительные свойства:

| Имя  | Значение                                 |
| ---- | ---------------------------------------- |
| min  | Минимальный угол (например -45)          |
| max  | Максимальный угол (например 45)          |
| axis | Вектор оси вращения (например [0, 1, 0]) |

⚠️ Важно:
ось задаётся в системе координат **Y-Up**, в отличие от Blender

---

## Коллизии (Collision Hulls)

Коллизионные меши:

- являются дочерними объектами
- имя должно заканчиваться на `$hull`

Требования:

- меш должен быть **выпуклым (convex)**
- должно быть **чётное число граней**

Иначе .sur не будет создан корректно

### Специальные случаи

- Щит корабля:

  ```
  HpMount$hull
  ```

- Переопределение хитбоксов оборудования:

  ```
  HpWeapon01$hull
  ```

---

## LOD (уровни детализации)

Формат имени:

```
<имя части>$lod1
<имя части>$lod2
```

Пример:

```
wing$lod1
wing$lod2
```

---

## Настройки импорта

- **Model Name**
  Имя выходного файла (без расширения)

- **Generate Materials**
  Генерация библиотек материалов и текстур

- **Import Textures**
  Импорт текстур из модели

- **Placeholder Textures**
  Генерация заглушек для отсутствующих текстур

- **Advanced Materials**
  Импорт PBR / normal map (экспериментально)
  ⚠️ Не совместимо с ванильным Freelancer

- **Generate sur**
  Генерация файла коллизий .sur

- **Output Path**
  Папка сохранения

- **Starsphere Mode**
  Принудительное создание .cmp вместо .3db
  (нужно для звёздных сфер)

---

## Имена материалов

Во вкладке **Material Names**:

- отображаются все используемые материалы
- можно переименовать перед экспортом

Это полезно для:

- подключения внешних .mat файлов
- избежания конфликтов имён
