# Model Importer

Импортер моделей LancerEdit поддерживает создание как файлов **.cmp**, так и **.3db** (соответственно для многосоставных и односоставных моделей) из распространённых форматов обмена моделями. Также при необходимости он создаёт файлы коллизий **.sur**.

Импортер моделей доступен через **Tools -> Import Model**.

![Menu Option](assets/modelimport-menu.png)

---

## Supported Formats

### glTF 2.0 (.glb/.gltf)

Формат glTF 2.0 полностью поддерживается импортером моделей LancerEdit. Использование экспорта в **.glb** (или встроенного .gltf из вашего 3D-редактора) гарантирует, что все ресурсы будут упакованы внутри файла glTF.

Это включает:

- текстуры
- пользовательские свойства

---

### Wavefront .obj (.obj)

**Не рекомендуется.** Поддерживает только создание односоставных моделей (.3db), пользовательские свойства и многосоставные модели этим форматом не поддерживаются.

---

### Collada (.dae)

Резервный формат импорта/экспорта на случай, если glTF 2.0 не поддерживается вашим инструментом моделирования. Пользовательские свойства в этом формате отсутствуют, поэтому hardpoints и составные соединения будут импортированы некорректно.

---

## Blender Integration

LancerEdit поддерживает открытие файлов из Blender, автоматически выполняя экспорт в glTF 2.0. LancerEdit попытается автоматически определить установленный Blender, но если это не удастся, вы можете указать путь вручную в настройках.

- Перейдите в **Tools -> Options** и укажите путь к исполняемому файлу Blender во вкладке Import/Export.

## Model Hierarchy

LancerEdit требует, чтобы корневой меш модели находился **в корне файла** — он станет:

- корневой частью в .cmp
- или самой моделью в .3db

Его нельзя размещать внутри пустого вспомогательного объекта (helper).

Дочерние элементы этого меша сохраняются как непосредственные потомки.

Проверить итоговую иерархию можно во вкладке **Output Nodes** импортерa.

![Model Hierarchy](assets/modelimport-hierarchy.png)

---

### Hardpoints

Hardpoints хранятся как дочерние объекты мешей с набором пользовательских свойств.

Обязательное свойство:

- `hardpoint` — должно быть задано и не равно 0 (это указывает импортеру, что объект является hardpoint)

Фиксированные hardpoints имеют свойство:

```
hptype = "fix"
```

Вращающиеся hardpoints имеют:

```
hptype = "rev"
```

И дополнительные свойства:

| Name | Value                                                                                                                                                                                                 |
| ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| min  | Минимальный угол в градусах (например -45)                                                                                                                                                            |
| max  | Максимальный угол в градусах (например 45)                                                                                                                                                            |
| axis | Массив из 3 чисел, задающий ось вращения (например [0, 1, 0]). <br><br> _Note: Это значение всегда задаётся в системе координат Y-Up. Blender и другие редакторы используют другую систему координат_ |

---

### Collision Hulls

Коллизионные меши:

- являются прямыми дочерними объектами родителя
- имеют имена, заканчивающиеся на `$hull`

Требования:

- меш должен быть **выпуклым (convex)**
- должен иметь **чётное число граней**

Иначе корректный .sur файл создан не будет.

Щит корабля:

- задаётся выпуклым мешем с именем `HpMount$hull`, прикреплённым к root

Override-хитбоксы оборудования:

- создаются аналогично
- имя должно совпадать с hardpoint

Пример:

```
HpWeapon01$hull
```

---

### LODs

LOD-меши могут находиться в любом месте файла и имеют имена вида:

```
<имя части>$lod1
```

Пример:

```
wing$lod1
wing$lod2
```

## Settings

![Importer Settings](assets/modelimport-settings.png)

- **Model Name**: Имя выходного файла модели (без расширения)
- **Generate Materials**: Создавать ли библиотеки текстур и материалов при импорте. Если отключено — все настройки текстур также отключаются
- **Import Textures**: Импортировать ли текстуры из модели. Отключается автоматически, если текстуры отсутствуют
- **Placeholder Textures**: Генерация заглушек, если текстура не найдена или не указана. Отключите, если используете внешнюю библиотеку текстур
- **Advanced Materials**: Импорт PBR-материалов и normal map из glTF или Blender. Экспериментальная функция, формат может измениться. Не совместимо с ванильным Freelancer
- **Generate sur**: Генерация файла коллизий .sur. Отключается, если нет коллизионных мешей. Файлы .sur сохраняются отдельно в Output Path
- **Output Path**: Папка для сохранения .cmp/.3db (и .sur при необходимости)
- **Starsphere Mode**: Включать для моделей, используемых как starsphere. Принудительно создаёт .cmp вместо .3db, так как .3db не работает для starsphere в ванильном Freelancer

## Material Names

![Material Names](assets/modelimport-materials.png)

В этой вкладке отображаются используемые материалы, и их имена можно изменить перед сохранением модели.

Это полезно, если:

- вы хотите использовать материалы из внешнего .mat файла
- нужно избежать конфликтов имён после экспорта модели
